# マイグレーション (Migration)

> 既存システムからの移行が不要な場合、本セクションはスキップしてください。

| 質問                                                                         | 質問の意図                                                                                   | ご回答 |
| ---------------------------------------------------------------------------- | -------------------------------------------------------------------------------------------- | ------ |
| 移行対象のシステム・サーバー・データベースの棚卸しは完了していますか？ | AWS Application Discovery Service や Migration Hub で移行対象を可視化し、依存関係を把握する。<br />AWS Transform（旧 Migration Evaluator）で AI 駆動のインフラ分析とビジネスケース生成を活用できる。<br />棚卸しが不十分だと移行漏れや想定外の依存関係の問題が発生する。 |        |
| 各システムの移行戦略（7Rs）は決定していますか？<br />・Rehost（リホスト）<br />・Replatform（リプラットフォーム）<br />・Refactor（リファクタリング）<br />・Repurchase（リパーチェス）<br />・Relocate（リロケート）<br />・Retain（リテイン）<br />・Retire（リタイア） | システムごとに最適な移行戦略を選択する。<br />ビジネス価値、技術的複雑さ、移行コストを総合的に判断する。 |        |
| データ移行の対象データ量と移行方式をご教示ください。 | データ量に応じて移行方式を選択する。<br />AWS Database Migration Service (DMS) でデータベースのオンライン移行が可能。<br />AWS DataSync でファイルデータの移行を高速化できる。<br />大量データ（数十TB以上）は AWS Snow Family（Snowball, Snowcone）での物理搬送も検討する。 |        |
| カットオーバー方式はどのように想定していますか？<br />（ビッグバン / 段階的移行） | ビッグバンはリスクが高いが移行期間が短い。<br />段階的移行はリスクを分散できるが並行運用が必要。<br />システム間の依存関係に応じて方式を検討する。 |        |
| 並行運用期間はどの程度を想定していますか？ | 移行元と移行先を同時に稼働させる期間を見積もる。<br />並行運用中のデータ同期方式（DMS CDC 等）やライセンスコストを考慮する。 |        |
| ロールバック計画と判断基準は定義されていますか？ | 移行に失敗した場合の切り戻し手順と判断基準を事前に定義する。<br />ロールバック可能な時間的制約（ポイント・オブ・ノーリターン）を明確にする。 |        |
| DNS やネットワークの切り替え方式は検討されていますか？ | Route 53 の加重ルーティングで段階的に切り替える方法もある。<br />DNS TTL の事前短縮、切り替え手順、切り戻し手順を設計する。 |        |
| 移行後の検証・受入テストの計画はありますか？ | 機能テスト、性能テスト、データ整合性チェックの計画を策定する。<br />テスト基準と合否判定条件を事前に合意しておく。 |        |
| 移行時に許容できるダウンタイムをご教示ください。 | 許容ダウンタイムに応じて移行方式を選択する。<br />ゼロダウンタイムが必要な場合は DMS CDC（Change Data Capture）や DRS による継続的レプリケーションを検討する。 |        |
| 移行スケジュールとマイルストーンは定義されていますか？ | 評価 → PoC → パイロット移行 → 本番移行 の段階を設定する。<br />ビジネス上のクリティカルな時期（決算期、繁忙期）を避けた移行スケジュールを策定する。 |        |
| 既存環境の廃止計画はありますか？ | 移行完了後の既存環境の廃止スケジュール、データ消去方法、ライセンス返却を計画する。<br />廃止までの期間のランニングコストも見積もる。 |        |
| 移行に伴うライセンスの変更影響を把握していますか？ | クラウド移行でライセンス体系が変わる場合がある。<br />AWS License Manager で BYOL ライセンスの追跡・管理ができる。<br />ライセンス販売元にクラウド利用の可否と条件を確認しておく。 |        |
