# サステナビリティ (Sustainability)

| 質問                                                                         | 質問の意図                                                                                   | ご回答 |
| ---------------------------------------------------------------------------- | -------------------------------------------------------------------------------------------- | ------ |
| AWS リージョンの選択においてサステナビリティを考慮していますか？ | AWS はリージョンごとに再生可能エネルギーの利用比率が異なる。<br />パフォーマンスやコストに加え、環境負荷も選定基準に含めることを検討する。 |        |
| リソース効率化の方針はありますか？<br />（Graviton プロセッサ / サーバーレス / マネージドサービスの活用） | Graviton ベースのインスタンスは同等性能で消費電力が少ない。<br />サーバーレスアーキテクチャはアイドル時のリソース消費をゼロにできる。<br />マネージドサービスの活用で共有インフラのリソース効率が向上する。 |        |
| 不要データの削除やアーカイブに関するポリシーはありますか？ | 不要なデータを保持し続けるとストレージコストとエネルギー消費が増加する。<br />S3 ライフサイクルポリシー、S3 Glacier、DynamoDB TTL 等を活用してデータを適切にアーカイブ・削除する。 |        |
| カーボンフットプリントの可視化・削減に取り組んでいますか？ | AWS Customer Carbon Footprint Tool で AWS 利用に伴う炭素排出量を可視化できる（Scope 3 排出データにも対応し、製造・輸送を含む包括的なカーボンフットプリントを把握可能）。<br />削減目標がある場合は、リージョン選択やアーキテクチャ設計に反映する。 |        |
| 組織としてサステナビリティに関する目標やKPIを設定していますか？ | 組織のサステナビリティ目標がある場合、IT インフラ設計にも反映する。<br />例: 2030年までに炭素排出量を xx% 削減、再生可能エネルギー比率 xx% 以上など。 |        |
