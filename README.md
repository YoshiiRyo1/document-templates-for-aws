AWS 上にシステムを設計・構築していくうえで役に立つドキュメントテンプレートを公開しています。  


## ディレクトリ構成

```
.
├── design/doc_source/    … クラウド設計シート（インフラ基本設計書テンプレート）
├── survey/doc_source/    … ヒアリングシート（非機能要件チェックリスト）
├── workshop/doc_source/  … ワークショップ資料（AWS ベストプラクティスの解説）
├── steering/             … 過去の対応計画・参考資料
├── .claude/              … Claude Code の設定（スキル・リファレンス）
├── CLAUDE.md             … Claude Code プロジェクト指示書
├── .mcp.json             … MCP サーバー設定
└── history.md            … 変更履歴
```

各ディレクトリの `README.md` にファイル一覧と詳細な説明があります。
ドキュメント本体は `doc_source/` 配下に格納されており、pandoc で MS Word へ変換して利用できます。

## Claude Code によるドキュメント管理

本リポジトリでは [Claude Code](https://docs.anthropic.com/en/docs/claude-code) を活用してドキュメントの作成・更新・レビューを行っています。
Claude Code は Anthropic が提供する CLI ツールで、AI がリポジトリの文脈を理解しながらドキュメント作業を支援します。

### 前提条件

- Claude Code がインストールされていること
- Python パッケージマネージャー `uvx` がインストールされていること（MCP サーバーの実行に必要）

### 設定ファイル

| ファイル                | 説明                                                                                                                                                 |
| ----------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------- |
| `CLAUDE.md`             | プロジェクト指示書。ディレクトリ構成、記述規約、ペルソナ、非機能要件の定義体系など、Claude Code がドキュメント作業を行う際のルールを記載しています。 |
| `.mcp.json`             | MCP（Model Context Protocol）サーバーの設定。AWS の最新ドキュメントや仕様を参照するための外部ツール接続を定義しています。                            |
| `.claude/settings.json` | Claude Code の権限設定。許可するツールや拒否するコマンドを定義しています。                                                                           |

### MCP サーバー

`.mcp.json` で以下の MCP サーバーが設定されています。

| サーバー名        | 用途                                                                                               |
| ----------------- | -------------------------------------------------------------------------------------------------- |
| `aws-knowledge`   | AWS ドキュメントの検索、リージョン別サービス提供状況の確認、ベストプラクティスの参照に使用します。 |
| `document-loader` | 外部ドキュメントの読み込みに使用します。                                                           |

### スキル（カスタムコマンド）

`.claude/skills/` 配下にプロジェクト固有のスキルが定義されています。
Claude Code のチャットで `/スキル名` と入力することで実行できます。

| スキル名             | 説明                                                                                                                                      | 実行方法              |
| -------------------- | ----------------------------------------------------------------------------------------------------------------------------------------- | --------------------- |
| `wa-review`          | AWS Well-Architected フレームワークの 6 本の柱に基づき、設計書やヒアリングシートをレビューして改善提案レポートを生成します。              | `/wa-review`          |
| `nfr-outdated-check` | 非機能要件ヒアリングシート（`survey/doc_source/nfr-*.md`）に記載された AWS サービスの仕様を最新情報と照合し、陳腐化した記述を特定します。 | `/nfr-outdated-check` |

| ファイル          | 説明                                                                                                                    |
| ----------------- | ----------------------------------------------------------------------------------------------------------------------- |
| `nfr-taxonomy.md` | 非機能要件の定義体系。IPA 非機能要求グレードと AWS Well-Architected フレームワークを統合した 7 カテゴリの分類定義です。 |
